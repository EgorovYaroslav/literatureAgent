# SciHubMCP — Автономный агент для обзора литературы

Автоматизированный пайплайн для обзора литературы, использующий Sci-Hub MCP + OpenCode для поиска, загрузки и организации научных статей.

---

## Предварительные требования

- Python 3.10+
- Установленный CLI [OpenCode](https://opencode.ai)
- Пакетный менеджер `uv` (`pip install uv` или `brew install uv`)
- Git

---

## Настройка

### 1. Установка OpenCode

```bash
# macOS / Linux
curl -fsSL https://opencode.ai/install.sh | sh

# Или через npm
npm install -g @opencode/cli
```

### 2. Запуск OpenCode Web UI

```bash
opencode web
```

Это откроет веб-интерфейс OpenCode в вашем браузере.

## Рабочий процесс (выполняется в OpenCode)

### Шаг A — Сброс к начальному состоянию

В чате OpenCode попросите агента:

> Запусти `python3 clear.py`, чтобы вернуть проект в исходное состояние.

Это сохранит только:
- `promt1.md`
- `promt2.md`
- `literature/2507.04211v1.pdf`
- `tools/scihub_tool.py`

### Шаг B — Запуск Промпта 1 (Настройка)

Попросите агента:

> Прочитай `promt1.md` и выполни его пошагово.

Это клонирует MCP-сервер Sci-Hub, создает виртуальное окружение с помощью `uv`, устанавливает зависимости, настраивает OpenCode MCP и создает `AGENT.md`.

### Шаг C — Запуск Промпта 2 (Уточнение)

Попросите агента:

> Прочитай `promt2.md` и выполни его пошагово.

Это инициализирует git, настраивает структуру проекта (`LIT.md`, `LIST.md`, `FOUND.md`, `diary.md`, `status.md`), отлаживает инструмент загрузки и переписывает `AGENT.md` со структурированным рабочим процессом.

### Шаг D — Выполнение обзора литературы

Попросите агента:

> Прочитай `AGENT.md` и выполни задачу пошагово, коммитя каждый шаг в git.

Агент выполнит:
1. Анализ исходного PDF → заполнение `LIT.md` → коммит
2. Определение 10 целевых статей → заполнение `LIST.md` → коммит
3. Поиск каждой в Sci-Hub → заполнение `FOUND.md` → коммит
4. Загрузка найденных PDF в `literature/` → коммит
5. Финальный обзор → коммит

## Ручное использование

```bash
# Активировать venv
source Sci-Hub-MCP-Server/.venv/bin/activate

# Поиск по DOI
python3 tools/scihub_tool.py search-doi "10.1093/mnras/sty2628"

# Поиск по ключевому слову
python3 tools/scihub_tool.py search-keyword "solar radio imaging" 5

# Загрузка PDF
python3 tools/scihub_tool.py download "<pdf_url>" "literature/paper.pdf"
```

## 🚀 Дополнительное задание: Кастомные Skills и Tools (Для самых смышленых)

В этом задании мы выйдем за рамки использования готовых инструментов и научимся **расширять возможности самого агента**.

Агент в OpenCode состоит из двух частей:
1. **Tools (Инструменты)** — "руки" агента. Функции, которые он может вызвать (например, `read_file`, `shell`, или наш `scihub`).
2. **Skills (Навыки)** — "мозг и опыт" агента. Это промпты-инструкции, которые говорят агенту, *как* и *в каком порядке* комбинировать Tools для решения сложных задач.

Ваша задача — добавить агенту новый **Навык (Skill)** для автоматического обзора литературы и написать недостающие **Инструменты (Tools)** для работы с PDF.

### Шаг 1. Добавление Skill (Навыка)

Навыки в OpenCode обычно хранятся в виде Markdown-файлов в специальной директории, чтобы агент мог их подгружать в свой системный промпт.

1. Создайте в корне проекта папку `skills/`.
2. Создайте файл `skills/literature.md`.
3. Скопируйте в него следующий код (это и есть описание навыка):

<details>
<summary><b>Нажмите, чтобы раскрыть содержимое skills/literature.md</b></summary>

```markdown
---
name: literature
description: Обзор литературы — web_search → pdf_download → анализ → literature/review.md
triggers: обзор литературы, scientific papers, arxiv, peer-reviewed, review
combines_with: markdown (для review.md), lean (при формализации найденных теорем), python (при анализе данных из работ)
---

# Обзор литературы — workflow

## Когда применяется
Пользователь просит:
- «Сделай обзор литературы по теме X»
- «Найди статьи про Y и проанализируй»
- «Скачай PDF и расскажи что там»

## Алгоритм
1. **Поиск кандидатов**: `web_search(query, num_results=10)`. Используй квалификаторы: `site:arxiv.org`, `filetype:pdf`.
2. **Уточнение у пользователя**: Покажи список и спроси, что скачивать. Не качай всё подряд без спроса!
3. **Подготовка папки**: `mkdir -p literature`.
4. **Скачивание PDF**: `pdf_download(url, dest_path)`. **Важно:** не используй `web_fetch` для PDF, он ломает бинарный контент.
5. **Глубокий анализ**: Используй `pdf_info`, `pdf_read`, `pdf_search`. Если работ >5, читай только первые 3 страницы.
6. **Отчёт**: Создай `literature/review.md` через `text_editor`. Структура: Список работ, Связи, Термины. **Не выдумывай цитаты!**
7. **Summary**: Коротко отчитайся в чат (сколько работ, путь к файлу, главные выводы).

## Правила
- Не выдумывай цитаты и номера страниц.
- Не изобретай связи между работами «по мотивам».
- Peer-reviewed фильтр: отмечай не-рецензируемые источники как «reference only».
```
</details>

### Шаг 2. Написание Tool (Инструмента) для работы с PDF

Если вы внимательно прочитали скилл, то заметили, что он опирается на инструменты `pdf_download`, `pdf_info`, `pdf_read` и `pdf_search`. Встроенных инструментов OpenCode для этого недостаточно (или они работают не так, как нужно).

**Ваша задача:** реализовать эти 4 функции.

#### Вариант А (Простой): CLI-обёртка
Создайте скрипт `tools/pdf_tool.py` (по аналогии с `scihub_tool.py`), который принимает аргументы командной строки:
```bash
python3 tools/pdf_tool.py download <url> <dest_path>
python3 tools/pdf_tool.py info <path>
python3 tools/pdf_tool.py read <path> <pages>
python3 tools/pdf_tool.py search <path> <query>
```
*Подсказка: используйте библиотеки `requests` для скачивания и `PyMuPDF` (fitz) или `pypdf` для парсинга PDF.*

#### Вариант Б (Продвинутый): Локальный MCP-сервер
Создайте отдельный MCP-сервер для PDF, чтобы агент вызывал эти функции нативно, без `shell`.
1. Создайте файл `pdf_mcp_server.py`.
2. Реализуйте в нём 4 функции (tools) с использованием `mcp` SDK.
3. Зарегистрируйте его в `opencode.json`.

---

### Шаг 3. Интеграция и конфигурация

Если вы выбрали **Вариант Б (MCP)**, вам нужно обновить `opencode.json`, добавив туда ваш новый сервер:

```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "scihub": {
      "type": "local",
      "enabled": true,
      "command": ["./Sci-Hub-MCP-Server/.venv/bin/python", "./Sci-Hub-MCP-Server/sci_hub_server.py"]
    },
    "pdf_tools": {
      "type": "local",
      "enabled": true,
      "command": ["./Sci-Hub-MCP-Server/.venv/bin/python", "./pdf_mcp_server.py"]
    }
  }
}
```
*(Не забудьте установить нужные зависимости в ваше `uv` окружение: `uv pip install pymupdf requests`)*.

### Шаг 4. Тестирование нового агента

1. Перезапустите OpenCode Web UI (`opencode web`), чтобы он подхватил новый `opencode.json` и скиллы.
2. Напишите агенту в чат триггерную фразу из скилла:
   > «Сделай обзор литературы по теме Physics-Informed Neural Networks для space weather forecasting. Найди 3 свежие статьи на arxiv».
3. Наблюдайте, как агент:
   - Вспомнит навык `literature`.
   - Найдет статьи через `web_search`.
   - Спросит у вас подтверждения (или скачает сразу, если вы попросили).
   - Вызовет ваши кастомные `pdf_*` инструменты.
   - Сгенерирует красивый `literature/review.md`.

**Критерии успешного выполнения:**
- [ ] Агент успешно вызывает кастомные PDF-инструменты без ошибок.
- [ ] Агент следует алгоритму из `skills/literature.md` (не начинает писать review.md до скачивания и анализа).
- [ ] В `review.md` нет "галлюцинаций" (выдуманных цитат), все номера страниц взяты из реального текста через `pdf_search`/`pdf_read`.
