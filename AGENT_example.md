## 3. Что ты ОБЯЗАН делать

### 3.1. Вести STATUS.md

После **каждого блока работы** (перед тем как закончить сессию) обновляй `statement/STATUS.md`:

```markdown
# STATUS.md

## Текущий этап
[Например: "Этап 2. Численный эксперимент"]

## Что сделано за последнюю сессию
- [список выполненных действий]

## Что делается сейчас
[текущая задача]

## Следующие шаги
[план на следующую сессию]

## Проблемы/блокировки
[если есть]
```

**Без STATUS.md после разрыва SSH-сессии ты забываешь контекст!**

### 3.2. Вести diary.md (append-only)

Каждое действие записывай в `statement/diary.md` в формате:

```markdown
## [YYYY-MM-DD HH:MM] Краткое описание действия

**Контекст:** почему это делается
**Результат:** что получилось
**Файлы:** какие файлы созданы/изменены
```

### 3.3. Делать git-снэпшоты

Перед длинными экспериментами:
```bash
git add .
git commit -m "checkpoint: описание состояния"
git tag snapshot_$(date +%Y%m%d_%H%M%S)
```

Используй **линейную историю** (никаких `git rebase`, `git push --force`).

## 5. Цикл работы (от литературы до отчёта)

### Этап 1. Работа с литературой

**Вход:** `literature/*.pdf`
**Выход:** `statement/LIT.md`

1. Прочитать PDF, извлечь ключевые тезисы
2. Классифицировать работы, найти gap analysis
3. Написать `LIT.md` с IEEE-цитатами

### Этап 2. Декомпозиция и матпостановка

**Вход:** `statement/MOTIVATION.md`, `statement/LIT.md`
**Выход:** Заметки в `statement/diary.md` (блоки M.0-M.9)

1. Выписать все обозначения
2. Сформулировать все предположения
3. Разбить на логические блоки

### Этап 3. Проверка гипотез

**Вход:** Математическая постановка
**Выход:** `statement/HYPOTHESIS.md` (список H1-H9)

1. Найти "дыры" в постановке
2. Сформулировать гипотезы H1-H9 с привязкой к экспериментам

### Этап 4. Постановка эксперимента

**Вход:** `statement/HYPOTHESIS.md`
**Выход:** `statement/TZ.md`

1. Написать TZ.md с техническими требованиями
2. Подготовить план эксперимента (сweep-сетка)

### Этап 5. Реализация эксперимента

**Вход:** `statement/TZ.md`
**Выход:** `experiment/plots/`, логи в `statement/diary.md`

1. Написать скрипт эксперимента (если ещё нет)
2. Запустить sweep (с фиксированным seed)
3. Сохранить графики в `experiment/plots/`
4. **Каждый sweep → запись в diary.md с привязкой к гипотезе**

### Этап 6. Интерпретация

**Вход:** Графики, логи
**Выход:** Заметки для автора

> ⚠️ **Интерпретация — работа автора, не агента!**
> Ты только предоставляешь данные. Вопрос "что значит этот результат?"
> решает исследователь.

### Этап 7. Написание статьи (LaTeX)

**Вход:** `statement/` (все md файлы) + `experiment/plots/`
**Выход:** `notes/paper/ai4math_ysda2026_template/example_paper.tex`

1. Взять математическую постановку из `statement/`
2. Добавить результаты экспериментов (графики из `experiment/plots/`)
3. Заполнить `example_paper.tex` по структуре шаблона
4. Обновить `example_paper.bib`

## 7. Конкретная задача: кросс-частотное обобщение СРГ

### Входные данные

**Вопрос:** Насколько хорошо модель классификации радиоизображений СРГ,
обученная на данных 3000 МГц, обобщается на данные других частотных
каналов (6000 МГц и 12200 МГц)?

**FTP-сервер (только 2025 год):**
```
3000 МГц:  https://ftp.rao.istp.ac.ru/SRH/SRH0306/cleanMaps/2025MMDD/srh_I_2025-MM-DDThh:mm:ss_3000.fit
6000 МГц:  https://ftp.rao.istp.ac.ru/SRH/SRH0612/cleanMaps/2025MMDD/srh_2025MMDDThhmmss_6000_I.fit
12200 МГц: https://ftp.rao.istp.ac.ru/SRH/SRH1224/cleanMaps/2025MMDD/srh_2025MMDDThhmmss_12200_I.fit

```

**Условие:** файлы должны совпадать в пределах 15 минут.  ⚠️

**Данные уже скачаны в:** `data/fits/SRH{channel}/`

### API для инференса

Использовать предоставленный скрипт `materials/api_example.py`:
```python
API_URL = "https://forecasting.iszf.irk.ru/api/srh/predict"

# Возвращает:
# {
#     "label": predicted_class,
#     "probability": confidence,
#     "image": base64_encoded_image
# }
```

### Эксперимент

**Задача:**
1. Провести **100 экспериментов**
2. Для каждого:
   - Случайно выбрать временное окно (2025 год)
   - Скачать 3 файла (3000, 6000, 12960 МГц) с разницей ≤ 3 мин
   - Получить предсказания модели для всех трёх каналов
3. Посчитать:
   - Долю совпавших классов в парах (3000↔6000) и (3000↔12960)
   - Гистограмму с разбивкой по классам и каналам

**Гипотеза:** если модель классифицирует одинаково во всех (или некоторых)
каналах, то обучение на каждом из них по отдельности не требуется.

**Выводы:**
- Пригодность модели для инференса в других каналах
- Экономия ресурсов на обучение новых моделей
